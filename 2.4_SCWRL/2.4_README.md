# 2.4 SCWRL

SCWRL predicts protein side-chain conformations while keeping the input
backbone fixed. In this repository it is an auxiliary dependency of pyDock3:
pyDock3 can call SCWRL3 when an input PDB contains incomplete side chains before
calculating binding energies.

SCWRL was developed in the Dunbrack laboratory at Fox Chase Cancer Center. It
is third-party software and is not distributed by this repository.

## Relationship to the repository workflow

The batch helper in
[`3.3.2_Computing_pyDock_Scores_for_a_Set_of_Complexes`](../3.3.2_Computing_pyDock_Scores_for_a_Set_of_Complexes/3.3.2_README.md)
creates pyDock `INI` files and runs:

```text
pyDock3 <configuration_name> bindEy
```

SCWRL3 only needs to be installed and configured once. During scoring, pyDock3
uses the executable recorded in its own `etc/pydock.conf` file whenever
side-chain reconstruction is required. You do not need to run SCWRL3 as a
separate step in this repository workflow.

For obtaining and configuring pyDock itself, see
[`2.3_README.md`](../2.3_pyDock_Scoring/2.3_README.md).

## Install the SCWRL3 copy bundled with pyDock3

The pyDock3 distribution supplied to licensed users includes the archive:

```text
scwrl3_lin.tar.gz
```

This bundled archive is the preferred source for the pyDock3 workflow. Here,
`PYDOCK` must be the directory that contains both `pyDock3` and
`scwrl3_lin.tar.gz`:

```bash
export PYDOCK="/absolute/path/to/pyDock3"

test -x "${PYDOCK}/pyDock3"
test -f "${PYDOCK}/scwrl3_lin.tar.gz"

cd "${PYDOCK}"
tar -xzf scwrl3_lin.tar.gz
cd 3.0
./setup
```

The archive creates a directory named `3.0/` containing the installer and the
resulting `scwrl3` executable. Run `./setup` only after placing this directory
in its final location. If the installed directory is moved later, rerun
`./setup` there before using SCWRL3 through pyDock.

## Configure pyDock3

Edit:

```text
<PYDOCK>/etc/pydock.conf
```

Under `[apps]`, set `SCWRL` to the absolute path of the generated executable:

```ini
[apps]
SCWRL=/absolute/path/to/pyDock3/3.0/scwrl3
```

Use the real expanded path in this file, rather than the literal string
`$PYDOCK`.

## Safe verification

These checks do not run pyDock or modify a structure:

```bash
test -x "${PYDOCK}/3.0/scwrl3" \
  && echo "SCWRL3 executable found"

grep -n '^SCWRL=' "${PYDOCK}/etc/pydock.conf"
```

Confirm that the path printed by `grep` names the same executable tested by the
first command. A standalone functional run, when needed, follows the SCWRL3
interface:

```bash
"${PYDOCK}/3.0/scwrl3" \
  -i input.pdb \
  -o output_with_rebuilt_side_chains.pdb \
  > scwrl3.log
```

This last command creates a new PDB and log; it is not required for every
repository run because pyDock3 handles reconstruction internally when needed.

## Platform compatibility

The Linux `setup` program and SCWRL3 payload in the bundled archive are legacy
32-bit x86 executables. They require an x86 Linux system capable of executing
32-bit programs. They do not run natively on ARM systems, and some current
64-bit Linux installations may require compatibility support from the operating
system.

This limitation is separate from any compatibility packages required by the
pyDock3 executable itself.

## If the bundled archive is absent

First confirm that the complete licensed pyDock3 distribution was unpacked. If
`scwrl3_lin.tar.gz` is genuinely absent, legacy SCWRL3 may need to be requested
from the [Dunbrack laboratory](https://dunbrack.fccc.edu/lab/scwrl) or accessed
through the [SBGrid SCWRL3 entry](https://sbgrid.org/software/titles/scwrl3),
subject to their current access and licensing conditions.

Do not replace SCWRL3 with SCWRL4 silently. SCWRL4 is the newer implementation,
but it is not documented as a drop-in replacement for the automatic SCWRL3
integration in pyDock3. The chapter describes SCWRL4 compatibility in the
context of the forthcoming pyDock4 distribution. Until using a pyDock release
that explicitly supports SCWRL4, keep pyDock3 configured with SCWRL3.
