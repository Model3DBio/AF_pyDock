# AlphaFold 3

AlphaFold 3 extends biomolecular structure prediction beyond protein-only
systems. It is designed to model interactions involving proteins, DNA, RNA,
small-molecule ligands and ions, and it can represent supported
post-translational and chemical modifications. These capabilities are relevant
to protein-protein and antibody-antigen complexes as well as interactions with
nucleic acids and other molecular partners.

This section is an orientation to the two available access routes. It does not
duplicate the complete AlphaFold 3 installation procedure maintained in the
external guide linked below.

---

## AlphaFold Server

[AlphaFold Server](https://alphafoldserver.com/) provides a web interface for
submitting prediction jobs without installing AlphaFold 3 or interacting with
the command line. It supports complexes containing proteins, DNA, RNA, selected
ligands and ions, and a defined set of protein and nucleic-acid modifications.

This route is useful for preparing and running a small number of jobs quickly.
Users should review the server interface, supported entity list, job limits and
current terms before submission. The server exposes a more restricted set of
ligands and covalent modifications than the local code.

Prepared protein-only server inputs for the case studies are available in:

```text
3.1_Generating_3D_Models_for_Protein_Protein_Complexes_with_AlphaFold/
└── json_files_for_alphafold3_server/
```

These JSON files still require manual submission and external computation on
AlphaFold Server.

## Local AlphaFold 3

Local execution provides greater control over job generation, repeated runs and
the molecular entities supported by the input format. It is therefore useful
when producing a larger model ensemble or when a job requires entities that are
not available through AlphaFold Server.

The official source code and installation documentation are maintained at:

* [google-deepmind/alphafold3](https://github.com/google-deepmind/alphafold3)

The model parameters are not included in this repository. Access must be
requested using the form linked by Google DeepMind in the official AlphaFold 3
repository, and the parameters must be obtained directly from Google.

For a Conda-based installation that avoids additional system-level package
installation, follow the separate guide maintained by this group:

* [AlphaFold3-Conda-Install](https://github.com/Model3DBio/AlphaFold3-Conda-Install)

That guide covers Miniconda setup, environment creation, dependencies,
databases, model configuration and execution. This repository assumes that a
working installation is already available before using
`3.1_Generating_3D_Models_for_Protein_Protein_Complexes_with_AlphaFold/run_alphafold3.sh`.

## Relationship to the repository workflow

Section 3.1 provides two AF3 routes:

* prepared JSON inputs for manual AlphaFold Server submission;
* `run_alphafold3.sh` for an existing local installation, followed by optional
  CIF-to-PDB conversion with `cif_to_pdb.py`.

Generated or downloaded models are organized under:

```text
Case_Studies_Included/<CASE>/AF3/
```

They can then be processed by the pyDock scoring workflow in section 3.3.2.
