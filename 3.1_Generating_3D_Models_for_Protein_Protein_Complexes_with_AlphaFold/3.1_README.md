# Generating 3D Models with AlphaFold

This directory contains two Bash workflows for generating protein-protein
complex models:

* `run_af2_multimer_and_relax.sh`: creates an input FASTA file, runs
  ColabFold / AlphaFold2-Multimer v1, v2 and v3, saves intermediate recycles,
  and launches GPU relaxation jobs with Greasy.
* `run_alphafold3.sh`: creates a protein-only AlphaFold 3 JSON input and runs
  a local AlphaFold 3 installation.

Both scripts are configured with environment variables and include a default
`4POU` example.

---

## Workflow Summary

```text
Sequences
  -> AF2-Multimer / AF3 input files
  -> AlphaFold model generation
  -> model folders for downstream pyDock scoring
```

For the AF2 workflow, the script also creates a Greasy task file for relaxing
intermediate recycled models.

---

## Script 1: `run_af2_multimer_and_relax.sh`

This script runs ColabFold with the three AlphaFold2-Multimer model versions:

* `alphafold2_multimer_v1`
* `alphafold2_multimer_v2`
* `alphafold2_multimer_v3`

It uses the following ColabFold options:

| Option | Purpose |
| --- | --- |
| `--save-recycles` | Save intermediate recycle structures. |
| `--model-type alphafold2_multimer_v*` | Run AF2-Multimer v1, v2 and v3. |
| `--re-cycle-early-stop-tolerance 0` | Avoid early stopping before the requested recycles. |
| `--rank multimer` | Rank models with the multimer score. |
| `--use-dropout` | Increase model diversity during inference. |
| `--num-recycle 20` | Run up to 20 recycling iterations. |
| `--amber` | Enable Amber relaxation. |
| `--use-gpu-relax` | Use GPU acceleration for relaxation. |

### AF2 input variables

| Variable | Default | Purpose |
| --- | --- | --- |
| `CASE` | `4POU` | Case identifier used for folders and file names. |
| `SEQ` | 4POU VHH/RNase A sequence | Protein sequences. Separate chains with `:`. |
| `GREASY` | `/path/to/software/greasy/` | Directory containing the `greasy` executable. |
| `GREASY_NWORKERS` | `4` | Number of parallel Greasy workers for relaxation jobs. |

### AF2 usage example

```bash
export CASE="MyComplex"
export SEQ="AAAAABBBB:CCCCDDDD"
export GREASY="/opt/greasy"
export GREASY_NWORKERS=2

bash run_af2_multimer_and_relax.sh
```

### AF2 generated files

From the current working directory, the script creates:

```text
${CASE}/
└── AF2/
    ├── ${CASE}.fasta
    ├── ${CASE}.colabfold.v1/
    ├── ${CASE}.colabfold.v2/
    └── ${CASE}.colabfold.v3/

${CASE}.colabfold.relax.greasy
```

The FASTA file contains a `>${CASE}` header and the sequence wrapped at 60
characters per line.

The relaxation task file contains commands equivalent to:

```bash
colabfold_relax --max-iterations 2000 --tolerance 2.39 \
  --stiffness 10.0 --max-outer-iterations 3 --use-gpu input.pdb output.pdb
```

## Script 2: `run_alphafold3.sh`

This script prepares and runs a local AlphaFold 3 job. It converts `SEQ` into
an AlphaFold 3 JSON request, assigning chain IDs automatically from `A` to `Z`,
then calls the configured AlphaFold 3 runner.

The automatic JSON generator currently supports protein sequences only. It does
not generate ligand, DNA, RNA, glycan or custom modification entries.

### AF3 input variables

| Variable | Default | Purpose |
| --- | --- | --- |
| `CASE` | `4POU` | Case identifier used for folders and file names. |
| `SEQ` | 4POU VHH/RNase A sequence | Protein sequences. Separate chains with `:`. |
| `MODEL_SEEDS` | `13441` | One seed or several comma-separated integer seeds. |
| `APPDIR` | `/home/user/Programs` | Base software directory. |
| `ALPHAFOLD3DIR` | `${APPDIR}/alphafold3` | AlphaFold 3 installation directory. |
| `HMMER3_BINDIR` | `${CONDA_PREFIX}/bin` | Directory containing HMMER executables. |
| `DB_DIR` | `${ALPHAFOLD3DIR}/public_databases` | AlphaFold 3 database directory. |
| `MODEL_DIR` | `${ALPHAFOLD3DIR}/models` | AlphaFold 3 model parameter directory. |
| `RUN_AF3` | `run_alphafold.py` | AlphaFold 3 runner command. |
| `BUCKETS` | AF3 default bucket list | Bucket sizes passed to AlphaFold 3. |

Before launching AlphaFold 3, the script checks that the HMMER binaries,
database directory and model directory exist.

### AF3 usage example

```bash
export CASE="MyComplex"
export SEQ="AAAAABBBB:CCCCDDDD"
export MODEL_SEEDS="13441,2024"
export ALPHAFOLD3DIR="/path/to/alphafold3"
export HMMER3_BINDIR="/path/to/conda/env/bin"
export DB_DIR="/path/to/alphafold3/public_databases"
export MODEL_DIR="/path/to/alphafold3/models"
export RUN_AF3="run_alphafold.py"

bash run_alphafold3.sh
```

### AF3 generated files

From the current working directory, the script creates:

```text
${CASE}/
└── AF3/
    ├── ${CASE}.json
    └── output/
        └── af3_run.log
```

The JSON file has this general structure:

```json
{
  "name": "MyComplex",
  "dialect": "alphafold3",
  "version": 1,
  "modelSeeds": [13441],
  "sequences": [
    {
      "protein": {
        "id": "A",
        "sequence": "AAAAABBBB"
      }
    },
    {
      "protein": {
        "id": "B",
        "sequence": "CCCCDDDD"
      }
    }
  ]
}
```

## Requirements

AF2 workflow:

* ColabFold: `colabfold_batch` and `colabfold_relax`
* Greasy job scheduler
* CUDA-capable GPU
* GNU Bash

AF3 workflow:

* Local AlphaFold 3 installation
* AlphaFold 3 databases and model parameters
* HMMER executables: `jackhmmer`, `nhmmer`, `hmmalign`, `hmmsearch`,
  `hmmbuild`
* Python available in the active environment
* GNU Bash

## Troubleshooting

| Issue | Likely cause |
| --- | --- |
| `colabfold_batch` or `colabfold_relax` not found | ColabFold is not installed or not active in `PATH`. |
| No GPU detected | CUDA is not installed, not visible or not available to the job. |
| Greasy does not start | `GREASY` does not point to the directory containing `greasy`. |
| AF2 relaxation worker count does not change | Set `GREASY_NWORKERS` before launching the script. |
| AF3 reports missing HMMER binary | `HMMER3_BINDIR` does not point to the active environment or system HMMER directory. |
| AF3 reports missing databases or models | `DB_DIR` or `MODEL_DIR` does not point to a valid AlphaFold 3 installation. |
| AF3 JSON generation fails | `SEQ` is empty, has more than 26 chains, or contains unsupported characters. |
