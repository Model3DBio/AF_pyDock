# ColabFold on a Local Machine with LocalColabFold

This directory provides `install_colabbatch_linux.sh`, an adapted installer for
running ColabFold and AlphaFold2-Multimer locally. It is based on the Linux
installer from [YoshitakaMo/localcolabfold](https://github.com/YoshitakaMo/localcolabfold),
and prepares the software environment and model parameters used by this
repository's local AlphaFold2 workflow.

This is a convenience installer, not an official ColabFold or LocalColabFold
release. Review the script and the upstream documentation before using it on a
shared workstation or cluster.

---

## What the installer does

The script:

1. Creates a `localcolabfold/` directory below the current working directory.
2. Uses an available Conda installation or installs an x86-64 Miniforge
   distribution under `localcolabfold/conda/`.
3. Creates or reuses a named Conda environment called `Alphafold2` with Python
   3.10, OpenMM 8.2.0, PDBFixer, Kalign, HH-suite and MMseqs2.
4. Installs ColabFold from its GitHub repository, the AlphaFold dependencies,
   JAX for CUDA 12, TensorFlow and `silence_tensorflow`.
5. Downloads the LocalColabFold `update_linux.sh` helper and the AlphaFold2
   model parameters.

> **Existing environment warning:** if an environment named `Alphafold2`
> already exists, the installer reuses it and installs or upgrades packages in
> that environment. Inspect it first if it contains unrelated work.

---

## Requirements

* Linux on x86-64 (`Miniforge3-Linux-x86_64.sh` is used when Conda is absent).
* GNU Bash and `wget`.
* Internet access to GitHub, Conda channels and Python package indexes.
* Sufficient disk space for the Conda environment and AlphaFold2 parameters.
* For GPU inference, an NVIDIA driver and GPU compatible with the installed
  CUDA 12 JAX packages.
* Optional: an existing Miniforge, Miniconda, Mambaforge or Anaconda
  installation.

The script does not install an NVIDIA driver or other system-wide GPU software.
The exact compatibility requirements can change with JAX and ColabFold; consult
the current [LocalColabFold](https://github.com/YoshitakaMo/localcolabfold) and
[ColabFold](https://github.com/sokrypton/ColabFold) documentation.

## Choose the installation location

The installation root is intentionally the directory from which the script is
launched, not the directory containing the script. For example, the following
installs into `$HOME/Programs/localcolabfold`:

```bash
mkdir -p "$HOME/Programs"
cd "$HOME/Programs"

bash /path/to/AF_pyDock/2.2.3_ColabFold_on_a_Local_Machine_with_LocalColabFold/install_colabbatch_linux.sh
```

If the script is launched from the repository root, it will instead create
`AF_pyDock/localcolabfold/`. Avoid that invocation unless this is the intended
destination.

---

## Installation

From the chosen installation directory, run:

```bash
bash /path/to/AF_pyDock/2.2.3_ColabFold_on_a_Local_Machine_with_LocalColabFold/install_colabbatch_linux.sh
```

The installation downloads software and model parameters and can take a
significant amount of time. Do not interrupt Conda or pip while it is modifying
the environment.

The resulting locations depend on whether Conda already existed:

* With an existing Conda installation, `Alphafold2` is created in that Conda
  installation, while weights and LocalColabFold auxiliary files remain under
  `<installation_root>/localcolabfold/`.
* Without Conda, Miniforge is installed under
  `<installation_root>/localcolabfold/conda/`, and the named environment is
  managed by that local Miniforge installation.

## Activate and verify the environment

When an existing Conda installation was used:

```bash
conda activate Alphafold2
colabfold_batch --help
```

When the script installed Miniforge locally:

```bash
source /path/to/localcolabfold/conda/etc/profile.d/conda.sh
conda activate Alphafold2
colabfold_batch --help
```

To inspect whether JAX can see the expected device:

```bash
python -c 'import jax; print(jax.devices())'
```

After activation, `colabfold_batch` and `colabfold_relax` can be used by the
scripts in
`3.1_Generating_3D_Models_for_Protein_Protein_Complexes_with_AlphaFold/`.

## Updates and reproducibility

The installer pins some core components, including Python 3.10, OpenMM 8.2.0,
HH-suite 3.3.0 and JAX 0.5.3, but it also installs ColabFold from the current
GitHub branch and upgrades other unpinned packages. Re-running it at a later
date can therefore produce a different environment.

For reproducible calculations, record the installation date, the ColabFold
version and the package versions in the `Alphafold2` environment. The included
`update_linux.sh` is obtained from LocalColabFold; review the current upstream
instructions before using it to update an existing installation.

## Troubleshooting

| Problem | Check |
| --- | --- |
| `wget is not installed` | Install `wget` with the system package manager. |
| Conda environment conflict | Rename or remove an unrelated `Alphafold2` environment before installation. |
| JAX reports only a CPU device | Check the NVIDIA driver and compatibility with the CUDA 12 JAX packages. |
| `colabfold_batch` is not found | Initialize the Conda installation and activate `Alphafold2`. |
| Parameter download fails | Check network access and rerun after confirming the partial download state. |
| Installer reports an upstream compatibility warning | Compare the installed ColabFold version with the current upstream requirements before running predictions. |
